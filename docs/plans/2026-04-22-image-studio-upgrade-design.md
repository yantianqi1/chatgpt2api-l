# Image Studio Upgrade Design

**目标**

补齐图像工作台的可用性缺口：默认模型改为 `gpt-image-2`，管理页支持配置单次产图张数，图像页顶部显示剩余额度，失败请求支持自动重试与手动整组重跑，并根据是否上传图片自动判断文生图或编辑图。

**范围**

- 后端继续使用现有 `image settings` 运行时配置，不新增静默降级逻辑。
- 前端图像页从“手动模式切换”改为“自动模式判断优先，手动展示当前模式”。
- 会话记录扩展为可重跑，失败状态需要保留足够上下文用于整组重新生成。

**方案**

1. 后端保持 `default_model`、`max_count_per_request`、`auto_retry_times` 三个运行时设置，并把默认模型固定为 `gpt-image-2`。
2. 设置页新增“图片生成设置”卡片，直接读写 `/api/image/settings`，其中“单次最多产图张数”约束图像页输入上限。
3. 图像页加载设置与账号额度，顶部新增标题+额度信息块；提交时根据是否存在参考图自动选择生成或编辑接口。
4. 会话数据补充可重跑元信息，结果区提供“重新生成”按钮；失败或部分失败时支持整组重跑，并在会话内替换为新的进行中状态。
5. 前端重试分两层：单张请求在前端按配置次数重试；后端账号池保留现有上游失败重试。

**异常处理**

- 额度获取失败仅显示 `—`，不伪造额度。
- 重跑和首次生成都直接暴露真实错误，不吞掉失败原因。
- 页面刷新中断的任务仍标记为失败，便于用户手动重新生成。

**验证**

- 后端配置单测覆盖默认值与持久化。
- 前端静态检查覆盖新增类型和组件改动。
